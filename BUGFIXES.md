# Bug Fixes

All bug fixes made to the ORCA toolchain subrepos prior to and since the
unified monorepo was created. Listed in reverse chronological order.

## ORCA/C (orca-c/)

| Commit | Fix |
|--------|-----|
| `228e9a5` | **GNO startup missing ~_BWSTARTUP3 call — argc always 0.** The `gnoStartup` code path in Native2.pas skipped `JSL ~_BWSTARTUP3` entirely, so `SystemUserID` never saved the kernel-passed command line (X:Y registers) into `~CommandLine`. Every GNO-linked program received `argc=0`. Restored the `JSL ~_BWSTARTUP3` call before `SetDataBank`. |
| `33b6f07` | **#pragma lint -1 preventing code generation.** When `lintIsError` was true and all errors were lint errors, the compiler still set `liDCBGS.merrf := 16` (fatal), causing the code generator to be skipped entirely and producing unlinkable objects. Restructured the Error procedure to separate lint errors from real errors. |
| `33b6f07` | **#pragma gnostartup not implemented.** The pragma was parsed but had no code generation effect. Added the `gnoStartup` flag and a dedicated code path in Native2.pas `RootFile` that generates a GNO-appropriate startup sequence (no `~_BWSTARTUP3`, GNO memory model). |
| `2f2be04` | **LF line ending handling corrupted source files.** GoldenGate's `FastFileGS` incorrectly collapsed consecutive LF characters (blank lines in Unix-style files) into a single CR during text conversion. This caused `#define` bodies ending with `\<LF>` followed by a blank line to absorb subsequent source lines. Fixed by re-reading raw bytes via `OpenGS`/`ReadGS` and doing byte-by-byte LF-to-CR conversion. |
| `a48185d` | **Ported sheumann tier-2 fixes (3 bugs).** (1) Spurious errors/miscompilation for functions returning function pointers — returned function-pointer parameters were appended to the outer function's parameter chain. (2) Enum constant name visible during its own value expression — `NewSymbol` was called before evaluating the value, violating C scoping rules. (3) Nil/garbage pointer dereferences in Parser.pas, Scanner.pas, and Symbol.pas caused by ORCA Pascal's lack of short-circuit evaluation. |
| `f2fc5cb` | **Ported sheumann tier-1 fixes (7 bugs).** (1) Null pointer dereference in K&R vs prototype parameter type checking. (2) Nil pointer dereferences from non-short-circuit boolean evaluation. (3) `unsigned short` not promoted to `unsigned int` in integer promotions. (4) Comma expression type was integer-promoted instead of the right operand's exact type. (5) 32-bit assignment codegen forced `onStack` instead of respecting caller's A_X preference. (6) Casts to void accepted in integer constant expressions. (7) K&R-style parameters without explicit declarations got no local label. |
| `ca37948` | **Varargs stack corruption with optimize bit 6.** When calling a variadic function with extra args and `#pragma optimize 72` (`strictVararg=false`), `op^.q` was set to `ord(hasVarargs and strictVararg)` which masked out `hasVarargs`. The callee cleaned only its declared fixed parameters, leaving extra variadic args unpopped and corrupting subsequent stack usage. Fixed Expression.pas to use `ord(hasVarargs)` and Gen.pas to allocate the `stackLoc` DP slot whenever `hasVarargsCall` is true. |
| `00b2376` | **Inline asm sep/rep mode flags not updating longA/longI.** Gen.pas ORs `asmFlag` ($8000) into opcodes for inline assembly, but Native2.pas compared against raw opcode constants. Every comparison in the immediate-mode handler failed, so `sep #$30` emitted a 2-byte operand instead of 1, and `longA`/`longI` were never updated after `sep`/`rep`. Fixed by masking off `asmFlag` before the comparison, matching the original Native.pas. |
| `510bca0` | **Indirect function pointer calls crashed (BRK 00).** OMF opcode 7 encodes shift direction by the sign of its 32-bit operand. The high word was always emitted as -1 (right shift), but `shiftLeft8` requires +8 (left shift), corrupting the ORA immediate in `GenCui`. Also added the missing `ShiftSize`/`subtract1` branch in `genAddress`. |
| `510bca0` | **Prefix-file macros not defined under GoldenGate.** `Header2.pas` had a stub `InitHeader` that never called `DoDefaultsDotH`, so `__appleiigs__` and `__GNO__` were never defined. Also added `ExpandPath` to Scanner.pas so the `13:` device prefix resolves correctly before `GetFileType`. |
| `0cca38e` | **Segment buffer overflow for large functions (Error 112).** Functions generating more than ~16KB of object code crashed with "compiler error". `blkcnt` and `segDisp` in CGC.pas were 16-bit integers that overflowed. The label table was a static 163KB array too large for the 65816 code bank. The output buffer was fixed at 16KB. Fixed by widening counters to `longint`, splitting the label table into four heap-allocated halves, and replacing the fixed buffer with a dynamic one that starts at 64KB and doubles on demand. |
| `4e75a29` | **CGC.macros truncated.** A prior commit accidentally truncated CGC.macros from 513 to 307 lines and moved the `lla` macro to the top of the file (before the `return` macro it depends on), breaking the build. Restored the full file. |
| `f3e7a3f` | **Spec.Conform tests failing due to include path resolution.** `GetLibraryName` hit a GoldenGate quirk where `GetFileInfoGS` returns SRC for nonexistent paths under `13:ORCACDefs:`. Added `#pragma libpath` and restructured `GetLibraryName` to check user lib paths before the default prefix. |

## ORCALib (orcalib/)

| Commit | Fix |
|--------|-----|
| `0b5bfa2` | **fgets ABI mismatch — 2-byte stack leak and crash.** GNO's `stdio.h` declares `fgets(char *, size_t, FILE *)` where `size_t` is 4 bytes. The assembly popped only 2 bytes for the `n` parameter, causing a 2-byte stack leak that corrupted the return address and crashed any GNO caller. |
| `f0f4ae7` | **~User_ID zero under GNO exec.** GNO exec passes the userID in the A register, but the linker startup clobbers A before `~C_StartUp` runs. Added a safety check: if `~User_ID` is still 0 when `~C_StartUp` enters, allocate one via `GetNewID`. Also removed the `puts` OOM message to avoid pulling in SysLib. |
| `efd2fb0` | **GetNewID toolbox calling convention wrong in ~C_StartUp.** The tool dispatch number was $0902 instead of $2003 (`MiscTool #$03, function $20`) and the result space was not pushed/popped per IIgs toolbox convention. |
| `c6cf89d` | **argv array not null-terminated on empty command line.** When `cLine` is NULL, `~C_StartUp` now zeroes both `argv[0]` and `argv[1]` so the array has a proper NULL sentinel. Expanded `targv` from 4 to 8 bytes to hold the terminator. |
| `90d5bb5` | **sys/cdefs.h macro conflicts with GNO/busybox.** Added `#undef` guards for macros that conflicted when GNO busybox headers were included after ORCA headers. |
